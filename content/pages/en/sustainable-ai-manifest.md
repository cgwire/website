# Sustainable AI Manifesto

AI is built on huge energy use, resource extraction, and underpaid click
workers. On the other hand, the software industry has massively bet on it, with
outstanding progress. The consequence is that a software vendor can't ignore it
anymore. But what we can do is choose how we integrate it. This manifesto lays
out the limits we set to make it viable in the long term.

## 1. Ethics

AI should be designed with respect toward its users.

- Features must empower production teams and augment their capacity
- They don't need to push toward consumption
- AI must be included only to solve an identified pain point
- No addition whose usefulness hasn't been validated must be included
- Limits like hallucination must be clearly mentioned

## 2. Resources

AI consumes a lot of energy, and we should limit this.

- Tools must allow tracking token consumption
- Carbon tracking should be included
- Selecting models, like local models, should be possible
- If not applicable, users must be informed about which model is used
- Lighter models should be prioritized

## 3. UX

AI features should be implemented like any other.

- An "AI" color, icon, assistant name, face, or magic metaphor to depict an AI feature must be avoided
- An off switch must be possible, with a confirmation displayed when activating it
- The same AI function should not appear twice
- The feature description must tell what it does, not what it evokes
- Pushy onboarding is not needed

This is how we envision AI: powerful additions at the service of their user.
